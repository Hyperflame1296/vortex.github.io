Below is a simple Vortex language example demonstrating how to import external functionality and use it inside an entry point.
In this case, we bring in the Node.js console object and print out a classic "hello world" message. 

                        
// Vortex language example
@require External::console // get the console object from Node
entry() {
    console.log('hello world') // logs 'hello world' into console
}

The snippet above shows two basic features.

  • The @require allows you to pull in external modules or objects.
  • The entry() function defines the starting point of your program.

To run this program, you'll need a .vortexsettings file, and an index.js file.
(the .vortexsettings file is required for this to run!)
We actually don't need any important data the .vortexsettings file right now, so just put this in it for now: 

                        
{
    "version": 1
}

And the contents of index.js will look something like this: 

                        
// index.js
let { VortexInstance } = require('@hyperflame1/vortex') // get Vortex
let vortex = new VortexInstance() // create an instance of Vortex

vortex.loadSettings() // load the settings file from earlier
vortex.interpret('path/to/helloWorld.vx') // path to your Vortex file

Now, with this file, you can do node index.js in the terminal, and it will run!
(Vortex is an interpreter language. But a compiler to JavaScript will exist one day, trust me!)
